Dictionary attack

February 16, 2019 | written by @Barbe_verte

This attack consists of recovering each line of the dictionary that is sent to the hash function passed as a parameter.
The hash value is then compared to each line of values to be broken. If the two values are the same, we display the value and the password found in the dictionary and decrements the number of passwords to find.
One stops when one has tested all the words of the dictionary or that one has broken all the values of hash.

"Divide and conquer"

February 16, 2019 | written by @Barbe_verte

For dictionaries, we use the "divide and conquer", which is simply to give a part of the work to each thread, which corresponds to the total work divided by the number of threads.
Specifically we share the region "mapped" dictionary in number of threads. This technique has the merit of being simple and effective, and we can implement it much more easily without openMP than for bruteforce.
This technique is effective here since the amount of work is the same for each thread, except for the last one which adds to its part the rest of the division.

Brute Force Parallelism

February 10, 2019 | written by @Barbe_verte

For the brute-force, the function being recursively for the moment it was difficult to parallel it effectively, since each state depends on the preceding recursive call.
It is therefore sufficient to give each size to a thread, which will eventually make all recursive calls following his, while others will do the same thing but for another size.
The problem here is that each thread does not have the same amount of work.
